# 物业维修派单与材料核销系统

## 一、项目用途

本系统是一个完整的物业维修管理平台，覆盖从住户报修到完工评价的全流程，实现工单流转、维修派单、材料核销和数据统计的数字化管理。系统支持四种角色：

| 角色 | 核心功能 |
|------|----------|
| **住户** | 提交报修申请、查看个人工单、服务评价 |
| **客服** | 受理报修、按楼栋/类型/紧急程度派单、改派 |
| **维修工** | 查看派单、到场签到、处理记录、材料使用、完工提交 |
| **管理员** | 维修看板统计、材料库存管理、服务评价、数据导出 |

### 核心业务约束
1. **重复报修拦截**：同一住户同一类型未关闭工单不允许重复提交
2. **片区限制**：维修工仅能接收负责楼栋的工单
3. **库存校验**：材料库存不足时不允许完工提交
4. **到场前置**：未签到到场不允许直接完工
5. **评价锁定**：完工7天后评价入口关闭，不可修改

## 二、启动方式

### 环境要求
- Node.js >= 18
- npm >= 9

### 安装依赖
```bash
npm install
```

### 启动开发环境
```bash
npm run dev
```
该命令会同时启动：
- 前端 Vite 开发服务器：http://localhost:5173
- 后端 Express API 服务：http://localhost:3001

Vite 已配置代理 `/api` 转发到后端 3001 端口。

### 其他命令
```bash
# 仅启动前端
npm run client:dev

# 仅启动后端
npm run server:dev

# TypeScript 类型检查
npm run check

# 生产构建
npm run build
```

## 四、验收路径

### 4.1 住户端流程
```
登录（住户）→ 提交报修 → 查看工单列表 → 查看工单详情 → 完工后评价
```
验收点：
- [ ] 报修表单包含楼栋/单元/房间/类型/紧急程度/描述
- [ ] 同类型未关闭工单重复提交时系统拦截
- [ ] 工单详情显示完整流转步骤、派单记录、材料清单
- [ ] 完工后7天内可评价，超过7天评价锁定

### 4.2 客服端流程
```
登录（客服）→ 查看工单列表 → 选择待受理工单 → 派单给对应维修工 → 已派单工单可改派
```
验收点：
- [ ] 工单按状态分类统计展示
- [ ] 派单弹窗仅显示负责该楼栋的维修工（片区过滤）
- [ ] 支持改派操作，改派记录写入派单日志

### 4.3 维修端流程
```
登录（维修工）→ 查看我的工单 → 点击处理 → 到场签到 → 填写处理说明 → 选择使用材料 → 提交完工
```
验收点：
- [ ] 未签到时完工按钮禁用
- [ ] 材料选择器实时显示库存，低库存高亮警告
- [ ] 材料库存不足时提交被拦截
- [ ] 完工后材料自动扣减并写入流水

### 4.4 管理后台
```
登录（管理员）→ 维修看板 → 材料库存 → 服务评价 → 数据导出
```
验收点：
- [ ] 看板展示统计卡片 + 楼栋分布柱状图 + 类型分布饼图 + 月度趋势折线图
- [ ] 材料库存支持入库、退库操作，出入库流水完整记录
- [ ] 低分评价（≤2星）可发起返工，工单重新进入处理流程
- [ ] 支持三种导出类型：服务质量/材料核销/服务评价，CSV带UTF-8 BOM
- [ ] 导出历史可查看和重复下载

### 4.5 异常与特殊场景
- [ ] **超时标记**：已派单24小时未到场、处理中48小时未完工自动标记超时
- [ ] **返工流程**：从评价页或详情页发起返工，工单回到处理中状态，reworkCount+1
- [ ] **材料退库**：多余材料可退回仓库，流水记录为 return 类型
- [ ] **数据持久化**：重启服务后工单状态、派单记录、材料扣减、评价结果、导出历史均不丢失

## 五、关键数据流转

### 5.1 工单状态机
```
待受理(pending)
    ↓ 客服派单
已派单(dispatched)
    ↓ 维修工到场签到
处理中(processing)
    ↓ 维修工完工提交
待评价(pending_evaluation)
    ↓ 住户评价 / 超7天自动关闭
已关闭(closed)

特殊流转：
  已关闭 → 处理中（发起返工，isRework=true）
  已派单 → 已派单（改派，记录dispatch_logs）
```

### 5.2 材料核销流程
```
维修工选择材料+数量
    ↓ 后端事务校验
库存充足？ ──否──→ 返回错误，拦截提交
    │ 是
    ↓
1. 更新 materials 表 stock 字段（扣减）
2. 插入 material_transactions 流水（type=use）
3. 更新工单 completeRemark、completeImages、completedAt
4. 工单状态变更为 pending_evaluation
    ↓
事务提交，全部成功或全部回滚
```

### 5.3 数据库表结构

| 表名 | 用途 | 关键字段 |
|------|------|----------|
| users | 用户表 | id, name, role, buildingIds(JSON) |
| work_orders | 工单表 | orderNo, status, buildingId, workerId, arriveTime, completedAt, isOverdue, isRework |
| dispatch_logs | 派单日志 | orderId, fromWorkerId, toWorkerId, operatorId, remark |
| materials | 材料表 | name, unit, stock, warningThreshold, price |
| material_transactions | 材料流水 | materialId, orderId, type(use/stock_in/return/adjust), quantity, beforeStock, afterStock |
| evaluations | 评价表 | orderId, rating, content, isLocked |
| export_history | 导出历史 | type, fileName, filters(JSON), status |

### 5.4 超时判定规则（动态计算）
- **派单超时预警**：status=dispatched 且 dispatchedAt < now-24h
- **处理超时标记**：status=processing 且 arriveTime < now-48h
- 超时状态通过 `checkOverdue()` 函数在每次查询时动态计算，不依赖持久化字段

### 5.5 评价锁定规则
- 工单 completedAt 时间起算 7 天（168小时）内允许评价
- 超过时间窗口后端 evaluateOrder() 拒绝请求，前端禁用评价UI
- isLocked 字段标记，超期自动锁定不可修改

## 六、技术架构

```
┌─────────────────────────────────────────────────────┐
│                      前端 (React 18)                 │
│  ┌──────────┐  ┌────────────┐  ┌─────────────────┐  │
│  │  UI组件  │→│ Zustand Store│→│  fetch API请求  │  │
│  └──────────┘  └────────────┘  └────────┬────────┘  │
└─────────────────────────────────────────┼───────────┘
                                          │ /api/*
┌─────────────────────────────────────────┼───────────┐
│                   后端 (Express 4)       │           │
│  ┌─────────┐  ┌────────────┐  ┌─────────▼────────┐  │
│  │  Routes │→│  Services  │→│ better-sqlite3 DB │  │
│  └─────────┘  └──────┬─────┘  └──────────────────┘  │
│                      │ 业务校验                      │
│                      ├─ 重复报修拦截                  │
│                      ├─ 片区限制校验                  │
│                      ├─ 库存事务校验                  │
│                      ├─ 到场前置校验                  │
│                      └─ 评价时间窗口校验              │
└─────────────────────────────────────────────────────┘
```

- **前端技术栈**：React 18 + TypeScript + Vite + TailwindCSS 3 + Zustand + React Router v7 + Recharts + lucide-react
- **后端技术栈**：Express 4 + TypeScript + better-sqlite3（同步SQLite）
- **数据库**：SQLite 文件持久化，存储路径 `./data/property_repair.db`，启用WAL模式和外键约束
- **状态持久化**：用户登录状态使用 zustand persist 中间件存 localStorage；业务数据全部持久化到SQLite
