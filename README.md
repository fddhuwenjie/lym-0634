# 物业维修派单与材料核销系统

任务类型：0-1代码生成

业务领域：全栈Web应用

修改范围：跨模块多文件

任务难度：中等偏难

题面：

做一个物业维修派单与材料核销系统，住户提交报修后客服按楼栋、类型和紧急程度派给维修工，维修工记录到场、处理、使用材料和完工照片说明。正常工单要能从待受理流转到已派单、处理中、待评价和已关闭，并把材料扣减写入流水。重复报修未关闭、维修工跨片区接单、材料库存不足、未到场直接完工、住户超时评价后再次修改都要限制。后台需要维修看板、材料库存、服务评价和质量导出，导出包含处理时长、材料明细、评价结果和异常原因。重启后工单状态、派单记录、材料扣减、评价结果和导出历史不丢失。验收还要覆盖改派、返工、材料退库和超时工单标记。
